En esta actividad se trabaj con la base de datos CA2022_Cultivos_Superficie_Produccion.csv, proveniente del Censo Agropecuario 2022 (INEGI). Esta base contiene informacin sobre las unidades de produccin agropecuarias y las superficies cultivadas bajo distintos sistemas de riego en Mxico.

El objetivo fue preparar esta base para aplicar dos metodologas de clasificacin: Linear Discriminant Analysis (LDA) y rboles de Decisin podados, con el fin de predecir el nmero de Unidades de Produccin Agropecuaria Activas (UP activas), categorizadas en altas o bajas segn la mediana de la distribucin.

Para ello, se realiz un proceso de limpieza y depuracin de los datos que incluy:

  • Eliminacin de columnas no relevantes (NOM_MUN, NOMBRE, ENTIDAD).

  • Eliminacin de valores nulos en la variable objetivo.

  • Tratamiento de outliers con el mtodo de Tukey, para reducir la influencia de valores extremos.

  • Eliminacin de colinealidad alta entre variables (correlaciones > 0.85).

  • Creacin de una variable binaria target_bin:

0 (bajo): municipios con menos UP activas que la mediana.

1 (alto): municipios con ms UP activas que la mediana.

De esta manera, se obtuvo un dataset limpio y balanceado, listo para dividirse en entrenamiento y prueba, y posteriormente aplicar los modelos de clasificacin.

import pandas as pd
import numpy as np

df = pd.read_csv("CA2022_Cultivos_Superficie_Produccion.csv", encoding="latin-1")

# Eliminar columnas no necesarias
df = df.drop(columns=["NOM_MUN", "NOMBRE", "ENTIDAD"], errors="ignore")
target = "Total UP agropecuaria activas"
df = df.dropna(subset=[target])

#  Outliers (Tukey) 
Q1 = df[target].quantile(0.25)
Q3 = df[target].quantile(0.75)
IQR = Q3 - Q1
lim_inf = Q1 - 1.5 * IQR
lim_sup = Q3 + 1.5 * IQR
df_tukey = df[(df[target] >= lim_inf) & (df[target] <= lim_sup)].copy()

# Colinealidad
base = df_tukey if 'df_tukey' in globals() else df
y = base[target].copy()
cols_excluir = {target, "UP Agricolas", "y_winz", "y_log", "es_outlier"} & set(base.columns)
X = (
    base.drop(columns=list(cols_excluir), errors="ignore")
        .select_dtypes(include="number")
        .copy()
)
#  Eliminar colinealidad alta 
corr_matrix = X.corr().abs()
upper = corr_matrix.where(np.triu(np.ones(corr_matrix.shape), k=1).astype(bool))
to_drop = [column for column in upper.columns if any(upper[column] > 0.85)]
X = X.drop(columns=to_drop, errors="ignore")

df_final = X.copy()
df_final[target] = y

# Convertir variable objetivo a binaria (alto / bajo)
df_final["target_bin"] = (df_final[target] > df_final[target].median()).astype(int)

from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    roc_auc_score, confusion_matrix, classification_report
)

def eval_model(nombre, modelo, Xte, yte):
    y_pred = modelo.predict(Xte)
    out = {
        "Modelo": nombre,
        "Accuracy": accuracy_score(yte, y_pred),
        "Precision": precision_score(yte, y_pred, zero_division=0),
        "Recall": recall_score(yte, y_pred, zero_division=0),
        "F1": f1_score(yte, y_pred, zero_division=0),
    }
    try:
        y_prob = modelo.predict_proba(Xte)[:, 1]
        out["ROC_AUC"] = roc_auc_score(yte, y_prob)
    except Exception:
        out["ROC_AUC"] = roc_auc_score(yte, y_pred)
    print(f"\n=== {nombre} ===")
    for k,v in out.items():
        if k != "Modelo":
            print(f"{k:9s}: {v:.3f}")
    print("\nReporte de clasificacin:")
    print(classification_report(yte, y_pred, target_names=["Bajo","Alto"], zero_division=0))
    return y_pred

def plot_confusion(y_true, y_pred, titulo="Matriz de confusin"):
    cm = confusion_matrix(y_true, y_pred)
    fig, ax = plt.subplots(figsize=(6,5))
    im = ax.imshow(cm, interpolation='nearest', cmap=plt.cm.Blues)
    ax.set_title(titulo)
    plt.colorbar(im, ax=ax, fraction=0.046, pad=0.04)
    tick_marks = np.arange(2)
    ax.set_xticks(tick_marks); ax.set_xticklabels(["Clase 0 (Bajo)","Clase 1 (Alto)"])
    ax.set_yticks(tick_marks); ax.set_yticklabels(["Clase 0 (Bajo)","Clase 1 (Alto)"])
    ax.set_xlabel("Prediccin"); ax.set_ylabel("Real")
    for i in range(cm.shape[0]):
        for j in range(cm.shape[1]):
            ax.text(j, i, str(int(cm[i, j])),
                    ha="center", va="center", color="black", fontsize=14)
    fig.tight_layout()
    plt.show()

Tras aplicar y evaluar los modelos de Linear Discriminant Analysis (LDA) y rbol de Decisin podado sobre el conjunto de datos agropecuarios, se observa que ambos alcanzaron un rendimiento global similar en trminos de exactitud, con valores cercanos al 65% en el conjunto de prueba. Esto indica que en promedio ambos algoritmos logran clasificar de manera correcta a dos de cada tres observaciones.

Sin embargo, al analizar mtricas adicionales y las matrices de confusin, se aprecia un comportamiento distinto en cada modelo:

  • El LDA mostr una precisin ms alta para la clase Alto (0.73), lo que significa que cuando predice un municipio como de alta actividad agropecuaria suele acertar. No obstante, su recall bajo (0.46) refleja que deja escapar una gran cantidad de casos que en realidad pertenecan a esta categora, priorizando la deteccin de la clase Bajo.

  • El rbol de Decisin podado, en contraste, logr un balance ms adecuado entre precisin y recall (ambos alrededor de 0.630.66). Esto le permiti identificar un mayor nmero de municipios con alta actividad (217 vs. 158 detectados por LDA), aun si sacrific algunos aciertos en la clase Bajo. Adems, present un mejor desempeo en F1 Score (0.643 vs. 0.564) y un AUC ligeramente superior (0.708 vs. 0.705), lo que respalda su mayor capacidad de clasificacin equilibrada.

Otro aspecto a resaltar es la interpretabilidad: mientras que el LDA se limita a una frontera lineal entre las clases, el rbol podado ofrece reglas claras basadas en umbrales de las variables seleccionadas, lo que facilita su comprensin y aplicacin en contextos prcticos (ejemplo: si el nmero de unidades con riego es menor a cierto valor, clasificar como bajo). Adems, en trminos de eficiencia computacional, la implementacin con poda mediante K-Fold fue mucho ms viable que el mtodo inicial de LOOCV, que resultaba demasiado costoso en tiempo de ejecucin.

En conclusin, aunque ambos modelos obtuvieron rendimientos globales similares, el rbol de Decisin podado resulta ms adecuado para esta tarea especfica, ya que combina una capacidad de clasificacin equilibrada entre las dos clases, mtricas ligeramente superiores y una mayor interpretabilidad. Esto lo convierte en una herramienta ms confiable y til para la identificacin de municipios con alta o baja actividad agropecuaria dentro del dataset.
